# Security Policy

We take security seriously and appreciate responsible disclosure.

## Reporting a Vulnerability

- **Do not file public issues for security bugs.**
- Email the maintainers at `security@techitfactory.example` or open a private security advisory in GitHub.
- Include detailed reproduction steps, affected versions or commits, and any known mitigations.

## Response Expectations

- We aim to acknowledge new reports within **3 business days**.
- You will receive status updates as we triage, validate, and remediate.
- Coordinated disclosure timelines will be discussed as needed.

## Scope

- Documentation content, example manifests, and automation scripts in this repository.
- No production clusters or external services are managed by this project.

## After a Fix

- Security fixes will be noted in release notes and relevant documentation.
- When possible, mitigations or workarounds will be documented alongside the fix.
